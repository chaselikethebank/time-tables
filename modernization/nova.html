Introduction to Nova UI

A modern, utility-first CSS framework designed for coexistence with Bootstrap 3.4.1.

Nova UI provides a set of modern styling utilities and components, all carefully namespaced with the nv- prefix to prevent conflicts with existing Bootstrap 3 classes.

It embraces CSS variables for easy customization, a mobile-first responsive approach, and a clean aesthetic with no shadows or outlines, relying instead on color and typography.

Core Principles

  • Unique Prefix: All classes start with nv-.
  • Utility-First: Compose interfaces with granular classes.
  • CSS Variables: Easy theming and customization.
  • Modern & Clean: No shadows/outlines, focus on color/typography.
  • Responsive: Mobile-first with extensive breakpoints.
  • Flexbox Grid: Modern layout capabilities.

Core Concepts

Prefixing

Every class in Nova UI starts with nv- (e.g., nv-container, nv-p-5, nv-btn-primary). This avoids naming collisions with Bootstrap 3 classes like .container, .p-5 (if you were using a later Bootstrap), or .btn-primary.

CSS Variables

Nova UI heavily relies on CSS Custom Properties (Variables) defined in :root. You can easily override these to customize the look and feel (colors, spacing, fonts, etc.). See the Customization section.

Responsive Design

The framework is mobile-first. Utilities and grid classes can be applied responsively using breakpoint infixes:

  • No infix: Applies to all screen sizes (e.g., nv-p-3)
  • sm: Applies from --nv-breakpoint-sm (576px) upwards (e.g., nv-p-sm-4)
  • md: Applies from --nv-breakpoint-md (768px) upwards (e.g., nv-d-md-flex)
  • lg: Applies from --nv-breakpoint-lg (992px) upwards (e.g., nv-col-lg-6)
  • xl: Applies from --nv-breakpoint-xl (1200px) upwards
  • xxl: Applies from --nv-breakpoint-xxl (1400px) upwards
  • xxxl: Applies from --nv-breakpoint-xxxl (1600px) upwards

No Shadows / Outlines

Following a modern, flat design principle, Nova UI avoids default box-shadows and outlines. Visual feedback (like focus states) is handled through changes in background color, border color, or text color. Ensure your customizations maintain accessible focus indicators.

Grid System

Nova UI includes a powerful mobile-first flexbox grid system for building layouts of all shapes and sizes. It's based on a 12-column system with multiple tiers, enabling complex responsive layouts.

How it Works

  • Rows (.nv-row) are wrappers for columns. They use negative margins to counteract the padding on columns and ensure content aligns correctly. They are flex containers.
  • Columns (.nv-col-*, .nv-col-sm-*, etc.) create the horizontal divisions. They have horizontal padding (gutter).
  • You must place column content directly within columns, and columns must be immediate children of a row.

Spacing Utilities

Assign responsive-friendly margin or padding values to elements using shorthand classes based on a numeric scale mapped to the --spacing-* variables.

Notation

The classes are named using the format: {property}{sides}-{breakpoint}-{size}

  • Property: m (margin), p (padding)
  • Sides:
    • t - top (e.g., nv-mt-3)
    • b - bottom (e.g., nv-mb-5)
    • s - start (left in LTR, right in RTL) (e.g., nv-ms-2)
    • e - end (right in LTR, left in RTL) (e.g., nv-pe-4)
    • x - horizontal (left and right) (e.g., nv-px-6)
    • y - vertical (top and bottom) (e.g., nv-my-1)
    • blank - all 4 sides (e.g., nv-p-7)
  • Breakpoint: blank, sm, md, lg, xl, xxl, xxxl (e.g., nv-mt-md-0)
  • Size: 0 to 13 (mapping to --spacing-* variables), or auto (for margin)

Customization

Customize Nova UI by overriding the CSS variables defined in the :root scope.

You can do this in your own CSS file that is loaded after nova-ui.css.

Example: Changing Primary Color and Base Radius

/* your-custom-styles.css */

:root {
  /* Change the base primary color to a purple */
  --nv-color-primary-lightest: #f3e5f5;
  --nv-color-primary-lighter: #e1bee7;
  --nv-color-primary-light: #ce93d8;
  --nv-color-primary-base: #9c27b0; /* New base purple */
  --nv-color-primary-dark: #7b1fa2;
  --nv-color-primary-darker: #6a1b9a;
  --nv-color-primary-darkest: #4a148c;

  /* Make the default radius sharper */
  --radius: 0.2rem;

  /* Adjust base font size */
  --nv-font-size-base: 15px;
}

Simply include this CSS file after `nova-ui.css`, and the framework's components and utilities will automatically use your new values.

If you inspect this documentation site, you'll see it uses the default variables. If you were to apply the CSS above, the primary buttons, links, and default border-radius would change.
